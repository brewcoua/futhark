# flux

GitOps entrypoint. `cluster.yaml` (the `FluxInstance` CR) is applied once by
`ansible/roles/flux_bootstrap` — Flux does not reconcile this directory itself, to avoid
watching its own bootstrap. Everything else under `flux/` (`infra/`, `nodes/`) IS
reconciled: each holds a `ks.yaml`, the Flux `Kustomization` CR that points Flux at the
matching repo-root directory (`../infra`, `../nodes`).

## Conventions

- **Every Flux `Kustomization` CR is a file named `ks.yaml`**, in a directory named for
  what it reconciles. The plain kustomize resource-list file — what `kustomize build`
  itself looks for — keeps its required name, `kustomization.yaml`. Those are the only
  two file names used for Kubernetes YAML in this repo; if you're naming something else,
  you're naming it wrong.
- **`postBuild.substitute`/`substituteFrom` only affects Kustomization-rendered output**,
  never Helm `values:` blocks. Don't reference `${VAR}` inside a `HelmRelease.spec.values`
  expecting substitution — it won't happen. Inside plain manifests reconciled by a
  Kustomization, escape a literal `$` as `$$` or it gets treated as a substitution.
- Ansible-bridged Secrets (git deploy key, Infisical machine identity, and anything
  future config bridging needs) are regenerated by the relevant `task ans:*` run, not by
  Flux — deliberately outside GitOps reach, since they exist to seed values Flux/apps
  need before they can resolve anything themselves.
- Domains are never bridged this way. Each app declares its own hostname next to its own
  manifests when it's added — there is no repo-wide `DOMAIN` value.

## Bootstrap sequence

Run by `ansible/roles/flux_bootstrap` (`task ans:k0s`):

1. Install the Flux Operator via Helm.
2. Apply the `flux-system/git-deploy-key` and `infisical-operator/infisical-machine-identity`
   Secrets, both resolved from Proton Pass.
3. Wait for the Flux Operator to be ready.
4. Apply `flux/cluster.yaml`.

## Phase 2

Apps land under `nodes/<hostname>.<workflow>/`, one directory per app, each with its own
`ks.yaml` + `app/` — see `nodes/README.md`.
